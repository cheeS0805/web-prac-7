Тема 7. Обробники подій jQuery

Теорія — Обробники подій

Обробники подій — це функції, які виконуються тільки після вчинення певних дій.

Основні методи:

МетодОпис
.on(event, fn)Встановлює обробник події на обрані елементи
.off(event, fn)Видаляє обробник події
.one(event, fn)Обробник спрацьовує лише один раз
.on(event, selector, fn)Делегування — обробник на батьківський елемент, фільтр за селектором

Функції-скорочення (shorthand):

СкороченняПодія
.click(fn)Клік мишею
.dblclick(fn)Подвійний клік
.mouseenter(fn)Курсор заходить на елемент
.mouseleave(fn)Курсор покидає елемент
.focus(fn)Елемент отримує фокус
.blur(fn)Елемент втрачає фокус (розфокусування)
.keydown(fn)Клавіша натиснута
.submit(fn)Форма відправлена

Завдання 2 — При наведенні на посилання дописується href

Прив'яжіть всім посиланням подію — при наведенні в кінець тексту дописується href в круглих дужках.

jQuery офіційний сайт
MDN Web Docs
GitHub 
$('.t2-link').mouseenter(function() {
    var href = $(this).attr('href');
    $(this).text( $(this).text() + ' (' + href + ')' );
});

Завдання 3 — Після першого наведення відв'язати подію

Доповнити завдання 2: після першого наведення на посилання слід відв'язати від нього подію, яка додає href.

jQuery (наведіть 1 раз)
MDN Web Docs (наведіть 1 раз)
GitHub (наведіть 1 раз) 
function addHref() {
    $(this).text( $(this).text() + ' (' + $(this).attr('href') + ')' );
    $(this).off('mouseenter', addHref); // відв'язуємо після першого разу
}
$('.t3-link').on('mouseenter', addHref);
Наведіть курсор — href додається лише один раз, повторні наведення не дають ефекту.

Завдання 4 — INPUT при втраті фокусу виводить value в #test

Прив'яжіть всім INPUT подію — при втраті фокусу кожен INPUT виводить своє value в абзац з id "test".

Тут з'явиться value поля після втрати фокусу

$('.t4-inp').blur(function() {
    $('#test').text( $(this).val() );
});

Завдання 5 — INPUT виводить value в Alert тільки при першому натисканні

Для всіх INPUT — при натисканні виводити value в Alert, але тільки за першим кліком. Повторне натискання реакції не викликає.

$('.t5-inp').one('click', function() {
    alert( $(this).val() );
});
Клікніть на будь-яке поле — Alert з'явиться лише один раз.

Завдання 7 — Клік (квадрат, 1 раз) + подвійний клік (подвоїти, 1 раз)

Дано абзаци з числами.

  • По кліку — число замінюється на свій квадрат, але тільки за першим кліком.
  • По подвійному кліку — число подвоюється, але теж тільки перший раз.

4

6

8

$('.t7-p').one('click', function() {
    var n = parseInt( $(this).text() );
    $(this).text( n * n );
});

$('.t7-p').one('dblclick', function() {
    var n = parseInt( $(this).text() );
    $(this).text( n * 2 );
});
Клік замінює число на квадрат (1 раз). Подвійний клік подвоює поточне число (1 раз).

Завдання на делегування через .on() — №1

Дано ul з кількома li. Кнопка додає новий li з текстом «пункт». При кліку на будь-який li — в кінець додається !. Це повинно працювати і для нових li. Подія має бути на ul (делегування).

  • Пункт 1
  • Пункт 2
  • Пункт 3
// Делегування — обробник на ul, фільтр 'li'
// Спрацьовує навіть для li, доданих після ініціалізації
$('#delList').on('click', 'li', function() {
    $(this).text( $(this).text() + '!' );
});

$('#addLiBtn').on('click', function() {
    $('#delList').append('<li>пункт</li>');
});

Таблиця з користувачами (ім'я і прізвище). Під таблицею — форма для додавання нового рядка. При кліку на будь-яку комірку — prompt для зміни її тексту. Подія має бути на table (делегування).

// Делегування — обробник на table, фільтр 'td'
$('#usersTable').on('click', 'td', function() {
    var newText = prompt('Змініть текст комірки:', $(this).text());
    if (newText !== null) $(this).text(newText);
});

$('#addUserForm').on('submit', function(e) {
    e.preventDefault();
    var first = $('#inpFirstName').val().trim();
    var last  = $('#inpLastName').val().trim();
    if (first || last) {
        $('#usersTable tbody').append(
            '<tr><td>' + first + '</td><td>' + last + '</td></tr>'
        );
        $('#inpFirstName').val('');
        $('#inpLastName').val('');
    }
});
Натисніть на будь-яку комірку таблиці для редагування. Нові рядки, додані через форму, також редагуються — завдяки делегуванню.
